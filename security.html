Security policy

Last updated: 2026-05-27

Responsible disclosure

If you've discovered a security issue in any of our products or on our website, please email tbernath@radiant-mpc.com with the details. Include reproduction steps where possible. We acknowledge new reports within five business days and aim to confirm remediation within thirty days for non-critical findings.

Please:

  • Test only on accounts you own.
  • Do not attempt to access, modify, or destroy any other user's data.
  • Do not run denial-of-service tests.
  • Give us a reasonable window to remediate before public disclosure.

Our machine-readable security contact is published per RFC 9116 at /.well-known/security.txt.

How our stack is hardened

  • Edge: All web traffic terminates HTTPS at Cloudflare with Universal SSL. HTTP requests are 301-redirected to HTTPS. HSTS is asserted on every response.
  • Static surface: The marketing site (radiant-mpc.com) is built from a public GitHub repository and served via GitHub Pages.
  • Customer portal & admin: Authentication runs in a Cloudflare Worker against a D1 database. Sessions use HttpOnly, Secure, SameSite=Lax cookies. Admin pages are additionally gated behind administrator login with forced initial-password change.
  • License keys: Issued via the admin console and signed with a Cloudflare-managed signing key. Revocation is enforced both server-side and via a public revocation list that licensed products poll.
  • PHI: No protected health information is accepted by our products. Users confirm de-identification before each upload; the products operate on de-identified DICOM only.
